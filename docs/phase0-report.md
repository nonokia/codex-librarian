# Phase 0 レポート — 評価ハーネスとベースライン計測

日付: 2026-07-10 / 対象: `nonokia/weather-you-travel`(25 files / 49 symbols / 254 edges)
成果物: `librarian eval`(ADR-4 の「評価レポートを出す CLI」)、正解セット 16 ケース、
決定的展開リトリーバのベースライン数値。

## 計測の定義

- **match 率 = micro recall**: 全ケースの期待文脈エントリのうち、リトリーバが予算内で
  実際に取得できた割合。**precision は報告しない** — 正解セットは網羅的キュレーションでは
  ないため「取得したが期待外」は誤りと判定できない。代わりにコスト(items / chars)を圧力
  指標として併記する。
- diff に含まれるシンボル(シード)自体は当たりに数えない。diff を見れば分かるものを
  取得しても retrieval の成果ではないため。
- 正解セットは実装(リトリーバ)を見ずに、コードリーディングと実コミット履歴
  (`note` に sha を記載)から「レビューに必要だった文脈」としてキュレーションした。
  現行リトリーバに取れないものも含む — その差分こそが改善対象。

## ベースライン(hops=2, budget=8000 chars, 重み: calls 1.0 / extends 0.9 / references 0.7 / imports 0.4, decay 0.65)

| 指標 | 値 |
|---|---|
| ケース数 | 16(target 式 15 + 実 diff 式 1) |
| **micro recall** | **69.6%**(32/46) |
| macro recall | 76.0% |
| 完全一致ケース | 8/16 |
| 平均取得 items / chars | 6.1 / 7,576(予算 8,000 にほぼ飽和) |

アーキテクチャの目標値との関係: Phase 2 終了時 60% 以上 → **ベースラインで既に超過**。
ただし対象が小規模リポジトリ 1 つ・16 ケースであり、この数字は「ハーネスが機能する」
ことの証明であって精度の主張ではない。Phase 4 目標 85% との差分が改善余地。

## 失敗分析 — 次に効くレバー(すべて計測で裏取り済み)

1. **ハブシンボルのシード時に予算が飽和する(wyt-012, recall 0)。**
   `App`(135 行)を変更すると、シードだけで予算の過半を消費。さらに 1-hop 候補が
   全員同スコア(calls×decay=0.65)で並び、貪欲詰めがコスト盲目なため
   「安い定数(MOCK_DELAY_MS 等)が入り、本命のコンポーネント/サービスが 26 件 elided」
   という逆転が起きる。→ 詰め込みは score/cost 比にする、同点タイブレークを
   エッジ種別・近接で決める、の 2 案。§8-3 の「詰め込み順序自体を学習対象にする」の
   最初の具体例。
2. **module シンボル(テストファイル丸ごと)が高コストで落ちる(wyt-001/002/009)。**
   テストはトップレベル実行なので calls エッジの起点が module になり、文脈に入れるには
   ファイル全体のコストがかかる。→ テストファイルは describe/it ブロックを
   シンボル化するか、抜粋(シグネチャ+該当 it のみ)でコストを下げる。
3. **props 経由の 2-hop(データフロー)が減衰で落ちる(wyt-006, wyt-008)。**
   `isValidFlightNumber ← App → FlightInput` のような「同じデータを触る隣人」は
   0.42 まで減衰し、予算競争に負ける。→ エッジ重みの学習(PatternCache)の出番。
   現時点ではチューニングしない(ADR-4: 変更はこの数字で正当化してから)。

## 正解セットの現状と成長計画

`eval/golden/weather-you-travel.json` に 16 ケース(§8 リスク 1 の「30 件で開始」には未達)。
対象も設計書の想定(著名 OSS)ではなくセッションでアクセス可能な自リポジトリ 1 つ。
ハーネス自体はリポジトリ非依存なので、(a) ケース追加、(b) 対象リポジトリ追加は
運用フェーズで漸増させる(§8 リスク 1 の緩和策どおり「運用中のログから半自動追加」が本命)。

## 再現手順

```bash
node bin/librarian.js index <weather-you-travel の path> --db wyt.db
node bin/librarian.js eval eval/golden/weather-you-travel.json --db wyt.db --pretty
```
